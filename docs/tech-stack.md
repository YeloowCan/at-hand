# 随取 AtHand - 技术选型（Tech Stack）

## 一、总体技术方案

本项目采用 **Expo + React Native + NativeWind**
构建跨平台移动应用，目标是实现：

-   快速开发（高迭代效率）
-   原生体验（接近 iOS / Android）
-   强安全（本地加密优先）

------------------------------------------------------------------------

## 二、核心技术栈

### 2.1 框架

-   Expo（React Native 运行时）
-   React Native（跨平台开发）

### 2.2 UI & 样式

-   NativeWind（Tailwind CSS for RN）
-   React Native Reanimated（动画）
-   Expo Router（路由管理）

### 2.3 状态管理

-   Zustand（轻量状态管理）

### 2.4 本地存储

-   expo-sqlite（结构化数据）
-   expo-secure-store（密钥存储）

### 2.5 加密

-   expo-crypto
-   自定义 AES-256 加密方案

------------------------------------------------------------------------

## 三、核心能力实现

### 3.1 离线优先

-   所有数据存储于本地 SQLite
-   不依赖服务器

### 3.2 数据加密

-   数据库内容加密
-   密钥存储在 SecureStore
-   用户密码派生密钥

### 3.3 生物识别

-   expo-local-authentication
-   支持 FaceID / 指纹

### 3.4 文件处理

-   expo-file-system
-   图片/PDF 本地存储

------------------------------------------------------------------------

## 四、项目结构建议

    src/
     ├── app/            # 页面（expo-router）
     ├── components/     # UI组件
     ├── features/       # 业务模块
     ├── store/          # Zustand
     ├── db/             # 数据库
     ├── utils/          # 工具函数
     └── hooks/          # 自定义hooks

------------------------------------------------------------------------

## 六、性能优化

-   列表虚拟化（FlatList）
-   首页缓存（最近使用）
-   避免重复渲染（memo / selector）
-   懒加载图片

------------------------------------------------------------------------

## 七、安全策略

-   本地数据加密（AES-256）
-   不使用云存储（默认）
-   剪贴板自动清除（可选）
-   防截屏（后续增强）

------------------------------------------------------------------------

## 八、开发规范

-   TypeScript 全量使用
-   ESLint + Prettier
-   组件函数式 + Hooks
-   原子化组件设计

------------------------------------------------------------------------
